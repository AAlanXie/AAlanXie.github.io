java语言的特点

  • 面向对象的(oop)
  • 强类型机制, 异常处理, 垃圾自动回收
  • 跨平台
  • 解释型

面向对象和面向过程的区别

面向过程

  • 面向过程是一种以事件为中心的编程思想，编程的时候把解决问题的步骤都分析出来，然后用函数将其解决
  • 例子
    • 五子棋 如下，一步步调用函数:
        1. 开始游戏
        1. 黑子先走
        1. 绘制图面
        1. 判断输赢
        1. 轮到白子
        1. 绘制图面
        1. 判断输赢
        1. 返回步骤2
  • 优点:
    • 流程化使得编程任务明确
    • 效率高
  • 缺点:
    • 需要深入思考
    • 代码复用性差
    • 后期维护难度大

面向对象

  • 将每个人和事物都看做对象，每个对象有自己的属性和行为。
  • 例子
    • 五子棋:
        1. 黑白双方
        1. 棋盘系统
        1. 规则系统
  • 优点:
    • 结构清晰, 程序是模块化和结构化，更加符合人类的思维方式；
    • 易拓展, 代码重用率高，可继承，可覆盖，可以设计出低耦合的系统
    • 易维护, 系统低耦合的特点有利于减少程序的后期维护工作量
  • 缺点:
    • 开销大，当要修改对象内部时，对象的属性不允许外部直接存取，所以要增加许多没有其他意义、只负责读或写的行为.
    • 性能低，由于面向更高的逻辑抽象层，使得面向对象在实现的时候，不得不做出性能上面的牺牲，计算时间和空间存储大小都开销很大

JDK, JRE, JVM

  • JDK: Java Development kit
    • 用于开发
    • JRE + JAVA开发工具(java, javac等)
  • JRE: Java Runtime Environment
    • 用于运行java程序
    • JVM + JAVA核心类库
  • JVM: Java Virtual Machine
    • 将字节码转换为特定机器代码，并提供了内存管理/垃圾回收等安全机制
  • 为什么要采用字节码呢?
    • 运行顺序:
      • java源代码 -> 编译器 -> jvm可执行的java字节码 -> jvm -> jvm中的解释器 -> 机器可执行的二进制机器码 -> 程序执行
    • 好处:
      • 解决了解释性语言执行效率低的问题，又保留了解释型语言可以移植的特点。

java面向对象编程的三大特性

修饰符

  • default: 默认访问修饰符，在同一包内可见
  • private: 在同一类内可见，不能修饰类
  • protected : 对同一包内的类和所有子类可见，不能修饰类
  • public: 对所有类可见

封装

  • 将一个对象的方法和属性私有化，同时可以提供一些给外界访问的方法

继承

  • 使用已经存在的类的定义作为基础建立新类的技术, 新类的定义可以增加新的数据或者新的功能，也可以使用父类的功能

多态

  • 多态就是指程序中定义的引用变量所指向的具体类型和通过该引用变量发出的方法调用在编程时并不确定，而是在程序运行期间才确定

接口(interface)和抽象类(abstract)

  • 区别
    • 抽象类可以提供成员方法的实现细节, 接口中只能存在public abstract方法
    • 抽象类中的成员变量可以是各种类型的, 接口中的成员变量只能是public static final类型的
    • 抽象类中可以有静态代码块和静态方法, 接口中不能有
    • 一个类只能继承一个抽象类, 一个类可以实现多个接口

int 和 integer 的区别

  • int是基础数据类型, integer是引用类型
  • 拆箱与装箱
    • 装箱: 将基础数据类型保存成为对象
    • 拆箱: 将对象拆解成为基础数据类型

equals 和 == 和 hashcode

  • ==
    • 基本数据类型的比较, 是比较值
    • 若是引用类型, 则比较内存中存放的地址
  • equals
    • object类中的equals方法用于判断对象的内存地址是否是同一个地址
    • 一般类覆盖了equals方法是通过判断内容是否相等来判断对象是否相同
  • hashcode
    • 计算对象实例的hashcode, 在对象散列的时候作为key存入。
  • equals和hashcode关系
    • equals相等的对象, hashcode也一定相同
    • hashcode不同, equals也一定不同
    • hashcode相同, equals不一定相同

java的内存回收机制

  • 为什么要进行内存回收?
    • C、C++等语言中，内存的分配和释放由程序代码来完成，容易出现由于程序员漏写内存释放代码引起的内存泄露，最终导致系统内存耗尽。
    • 内存泄漏:
      • 内存泄漏（Memory Leak）是指程序中已动态分配的堆内存由于某种原因程序未释放或无法释放，造成系统内存的浪费，导致程序运行速度减慢甚至系统崩溃等严重后果。

内存标记方式

  • 引用计数法
    • 当对象被创建并赋值给变量的时候计数为1; 当有其他变量引用该对象的时候, 引用计数+1; 当引用对象的变量超出存活范围或释放对象引用的时候, 引用计数-1。对象引用计数为0的时候代表可以被垃圾收集器回收。
  • 根搜索算法
    • 从一些指定的根对象出发，遍历找到和这个根对象有引用关系的对象, 从这些对象开始继续寻找，从而形成一个个的引用链, 不在引用链上的对象标记成为不可达的对象, 也就是需要回收的对象。

内存回收方式

  • 标记-清理算法(mark and sweep)
    • 方法: 采用跟踪算法标记内存对象后, 扫描堆内存中未被标记的对象, 进行回收。算法不移动对象, 对不存活对象进行回收。
    • 造成的问题: 会引起内存碎片
  • 标记-整理算法(compacting)
    • 方法: 和标记-清理方法相同, 但是会把存活的对象向内存前半部分空闲区域移动, 并更新对象的指针
    • 造成的问题: 执行成本较高, 但是可以解决内存碎片的问题
  • 复制-清除算法(copying)
    • 该算法把内存分为空闲区和对象区，新建对象存储到对象区中。当对象区满时，先采用跟踪算法对对象进行标记，再把存活对象拷贝到空闲区，清空原对象区，空闲区和对象区互换角色。在拷贝过车中，程序需要暂停，此算法适用于存活对象叫少的情况，可以解决内存碎片问题。
    • 造成的问题: 把内存分为两部分, 会有一部分没法使用
  • 分代回收算法
    • 会根据对象的存活周期将内存区域分为三大块: 年轻代(Eden区, S0, S1), 老年代, 永久代
      1. 对象首次创建进行内存分配的时候，首先会放置在 Eden 区，当 Eden 区放满了或者当该对象太大无法放进 Eden 区的时候，此时会对年轻代（Eden区 和 S0）进行一次 GC，将幸存下来的对象放置在 S1，然后清空掉 Eden区和 S0 区；（此时年轻代采用的是 复制算法）
      1. 在上面第一步中对年轻代进行垃圾回收的时候，同时会对幸存的对象进行标记，统计每个幸存对象经历的 GC 次数；
      1. 当 S1 区满了之后，或者年轻代的对象经历过指定次数的 GC 之后，这部分对象会被放置到老年代之中；
      1. 当老年代也满了之后，便会对老年代进行一次 GC；（老年代采用的是 标记-整理算法）

垃圾回收器

  • 串行垃圾回收器（Serial Garbage Collector）；
    • 串行垃圾回收器会暂停所有的应用程序线程，并采用单独的的线程进行 GC。
    • 适用于单 CPU、并且对应用程序的暂停时间要求不高的情况，所以不太适合当前的生产环境。
  • 并行垃圾回收器（Parallel Garbage Collector）；
    • 并行垃圾回收器是 JVM 默认的垃圾回收器，相较于串行垃圾回收器而言性能稍有提升，它也是需要暂停所有的应用程序线程的，但是区别是它会使用多线程进行 GC。
    • 所以并行垃圾回收器适用于多 CPU 的服务器、并且能接受短暂的应用暂停的程序
  • 并发标记扫描垃圾回收器（CMS Garbage Collector）；
    • CMS 回收器也是一种并行的垃圾回收器，它会采用多线程来进行扫描堆内存，标记需要清理的对象并将这些对象清理掉。
    • 但是 CMS 它需要更多的 CPU 来保证程序的吞吐量，并且它保证了最短的回收停顿时间，所以，在服务器允许的情况下，为了达到更到的性能，我们应该使用 CMS 来代替默认的 并行垃圾回收器。
  • G1垃圾回收器（G1 Garbage Collector）
    • G1 垃圾回收器是在 JDK1.7 中才正式引入的一款垃圾回收器，“科技在进步，所以一般越是先进的技术一般会更好用并且会替代陈旧的技术”，好了，玩笑归玩笑，但是 G1 的引入，目的就是为了取代 CMS 的。
    • 不要被上面 G1 的示意图误导， G1 并没有将内存进行物理划分，它只是将堆内存划分为一个个的 Region，但是也是属于分代垃圾回收器，G1 仍然会区分年轻代和老年代，并且年轻代仍然会有 Eden 区和 Survivor 区。
