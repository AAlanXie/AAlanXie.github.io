进程, 线程, 协程

进程与线程的区别

  • 功能上: 进程是资源调度的基本单位, 线程是程序执行的基本单位
  • 内存上: 进程拥有独立内存, 线程共享内存
  • 开销上: 进程系统开销较大, 线程较小
  • 通信上: 进程间通信较为复杂, 线程因为有共享空间, 所以通信较为简单

协程与线程的区别

  • 协程又叫微线程
  • 线程与进程都是同步机制，协程是异步机制
  • 协程是抢占式的，线程是非抢占式的。协程需要用户释放使用权切换到其他协程，同一时间只有一个协程拥有使用权，相当于单线程

进程和线程的切换流程

  • 进程
      1. 切换页表使用新的地址空间，处理器中所有已经缓存的内存地址都作废
      1. 切换内核栈和硬件上下文
  • 线程的切换第一步是不需要做的，因为线程是共享所在进程的虚拟地址空间的

进程间的通讯

  • 进程间通信: Inter Process Communication(IPC)
    • 是一种允许进程间交流和同步动作的机制
  • 一般进程有两种类型
    • 独立进程
    • 合作进程
  • 通讯方式
    • 管道:
      • 简介: 管道是一种半双工的通信方式，数据只能单向流动，而且只能在具有亲缘关系的进程间使用。通常是父子进程关系。
      • 限制: 速度慢，容量有限
      • 分类:
        • 匿名管道
          • 有固定的读端和写端
          • 只能用于具有亲缘关系进程之间的通信
        • 命名管道
          • 可以在无关进程之间交换数据
    • 共享内存(shared memory)
      • 概念:共享内存就是映射一段能被其他进程所访问的内存，这段共享内存由一个进程创建，但多个进程都可以访问。共享内存是最快的 IPC 方式，它是针对其他进程间通信方式运行效率低而专门设计的。它往往与其他通信机制，如信号两，配合使用，来实现进程间的同步和通信。
      • 优点: 能够容易控制容量，速度快
      • 限制: 需要保持同步
    • 信号量:
      • 信号量是一个计数器，可以用来控制多个进程对共享资源的访问。它常作为一种锁机制，防止某进程正在访问共享资源时，其他进程也访问该资源。因此，主要作为进程间以及同一进程内不同线程之间的同步手段。
      • 限制: 不能传递复杂消息，只能用来同步
    • 信号:
      • 信号是一种比较复杂的通信方式，用于通知接收进程某个事件已经发生。
    • 消息队列: 容量收到系统限制
      • 消息队列是由消息的链表，存放在内核中并由消息队列标识符标识。消息队列克服了信号传递信息少、管道只能承载无格式字节流以及缓冲区大小受限等缺点。
    • socket: 任何进程间都能通讯，但是速度慢
      • 套解口也是一种进程间通信机制，与其他通信机制不同的是，它可用于不同及其间的进程通信。

进程间的同步方式

  • 临界区: 通过对多线程的串行化来访问公共资源或一段代码，速度快，适合控制数据访问
  • 互斥量: 协调对一个共享资源的单独访问
  • 信号量: 控制具有有限数量用户资源
  • 事件: 用来通知线程一些事件的发生，从而启动后继任务

进程的调度策略

  • 先来先服务: 谁先来谁先调度
  • 短作业优先: 按估计运行时间最短的顺序进行调度
  • 最短剩余时间: 按照剩余时间的顺序进行调度
  • 时间片轮转: 每个进程执行一个时间片，执行完成后放到就绪队列的末尾
    • 时间长度设置过长或者过短会有什么问题呢？
      • 过长会导致对于短的请求响应时间变长
      • 过短会导致进程切换次数过多，减低cpu效率
  • 优先级调度: 为每一个进程设置优先级, 防止优先级低的进程永远等不到调度，可以随着时间的推移增加等待进程的优先级

进程状态

  • 就绪: 进程已处于准备运行的状态，即进程获得了除CPU之外的一切所需资源, 一旦得到CPU即可运行。
  • 运行: 运行状态就是进程正在CPU上运行
  • 等待: 进程正在等待某一事件而暂停运行，比如等待某资源为可用或等待I/O完成
  • 有的地方可以加上创建和终止

线程的分类

  • 内核级线程: 创建，撤销，切换都由内核实现
  • 用户级线程: 不依赖操作系统核心，由线程库来完成创建和管理

线程安全

  • 什么是线程安全
    • 指某个函数在多线程执行的环境中，能够正确的处理执行者之前的公用变量，是的程序能够正常的完成
  • 如何解决
    • 加锁呗
      1. 同步代码块
      1. 同步方法
      1. 加锁

线程共享什么，独占什么

  • 共享: 共享进程的地址空间
    • 进程代码段
    • 进程的共有数据
    • 进程打开的文件描述符
  • 独占: 每个线程有独立的堆和栈
    • 线程id
    • 寄存器组的值
    • 堆栈
    • 优先级

堆和栈

  • 栈
    • 由操作系统自动分配释放 ，用于存放函数的参数值、局部变量
  • 堆
    • 堆由开发人员分配和释放， 若开发人员不释放，程序结束时由 OS 回收，分配方式类似于链表
  • 区别
    • 管理方式不同: 栈由操作系统自动分配释放， 堆的申请和释放由程序员控制，容易内存泄漏
    • 空间大小不同: 每个进程拥有的栈大小要远远小于堆大小
    • 分配效率不同: 栈由操作系统自动分配, 会在硬件层级对栈提供支持, 分配专门的寄存器存放栈的地址，效率较高。堆是c++提供的库函数进行申请和管理，实现较为复杂，效率比较低

内核态与用户态

  • 内核态
    • 在内核模式下，执行代码可以完全且不受限制地访问底层硬件。 它可以执行任何 CPU 指令并引用任何内存地址。 内核模式通常保留给操作系统的最低级别、最受信任的功能。 内核模式下的崩溃是灾难性的； 他们将停止整个 PC。
  • 用户态
    • 在用户模式下，执行代码无法直接访问硬件或参考内存。 在用户模式下运行的代码必须委托给系统 API 来访问硬件或内存。 由于这种隔离提供的保护，用户模式下的崩溃总是可以恢复的。 在您的计算机上运行的大部分代码都将在用户模式下执行。
  • 内核态和用户态切换的开销大
    • 保留用户态现场（上下文、寄存器、用户栈等）
    • 复制用户态参数，用户栈切到内核栈，进入内核态
    • 额外的检查（因为内核代码对用户不信任）
    • 执行内核态代码
    • 复制内核态代码执行结果，回到用户态
    • 恢复用户态现场（上下文、寄存器、用户栈等）
  • 什么情况会导致用户态到内核态切换
    • 系统调用：用户态进程主动切换到内核态的方式，用户态进程通过系统调用向操作系统申请资源完成工作，例如 fork() 就是一个创建新进程的系统调用，系统调用的机制核心使用了操作系统为用户特别开放的一个中断来实现，如Linux 的 int 80h 中断，也可以称为软中断
    • 异常：当 C P U 在执行用户态的进程时，发生了一些没有预知的异常，这时当前运行进程会切换到处理此异常的内核相关进程中，也就是切换到了内核态，如缺页异常
    • 中断：当 C P U 在执行用户态的进程时，外围设备完成用户请求的操作后，会向 C P U 发出相应的中断信号，这时 C P U 会暂停执行下一条即将要执行的指令，转到与中断信号对应的处理程序去执行，也就是切换到了内核态。如硬盘读写操作完成，系统会切换到硬盘读写的中断处理程序中执行后边的操作等

死锁

死锁的概念以及产生的条件

  • 概念
    • 在两个或多个并发的进程中，每个进程持有某种资源而又等待其他进程释放它或他们保持着的资源，在没有改变这种状态之前都不能向前推进的状态
  • 产生的条件
    • 互斥: 一个资源一次只能被一个进程使用
    • 请求与保持: 一个进程因为请求资源阻塞的时候，对已经获得的资源保持不放
    • 不可剥夺: 进程获得的资源，没有使用完之前不能强行剥夺
    • 循环等待: 若干进程形成头尾相接的环形等待资源关系

如何处理死锁

  • 不处理
  • 检测死锁并恢复
  • 资源动态分配以避免死锁
  • 破除四个死锁的必要条件之一来防止死锁

分页与分段

分页

  • 分页
    • 把内存空间分成大小相同的固定的块。因为程序数据存储在不同的页面中，需要一个页表来记录映射关系，实现从页号到物理块号的映射
    • 两次访问内存: 1. 从内存访问页表 2: 根据第一次的物理地址访问内存取出数据

什么是交换空间

  • 当内存资源不足的时候，linux把某些页的内容移到硬盘上的一块空间上，释放内存空间。硬盘上的就叫做交换空间。物理内存和交换空间的总容量就是虚拟内存的可用容量
  • 用途
    • 物理内存不足的时候可以将不常用的页交换出去
    • 程序启动的时候内存页被初始化，之后不再需要

页面替换算法

  • 运行过程中要访问的页面不在内存中，需要将其调入内存中
    • 最佳算法
      • 所选择的页将是最长时间不再被访问(理想)
    • 先进先出
      • 换出的页是最先进入的页面
    • LRU
      • 将最近最久未使用的页面换出
    • 时钟算法
