5. Longest Palindromic Substring

  • Description:
    • Given a string s, return the longest palindromic substring in s
  • Solutions:
    • one: dynamic algorithm
      • F(i, j) -> F(i+1, j-1) + compare(a[i], a[j]) and j > i.
      • i -> i + 1: bottom to top
      • j -> j - 1: left to right

10. Regular Expression Matching

  • Description:
    • Given an input string s and a pattern p, implement regular expression matching with support for ‘.’ and ‘*’ where:
      • ‘.’ Matches any single character.​​​​
      • ‘*’ Matches zero or more of the preceding element.
    • The matching should cover the entire input string (not partial)
  • Solutions:
    • one: Dynamic Algorithm
      • dp(i, j) means if s[0:i] and p[0:j] could match or not
      • if j == ‘.’ => we need to know if dp[i-1][j-1] is True or not
      • if j == letter => we need to know dp[i-1][j-1] and compare s[i] and p[j]
      • if j == ‘*’
        • dp[i-1][j] and (s[i] == p[j-1] or p[j-1] == ‘.’)
        • dp[i][j-2] or dp[i][j-1]

  • Solutions:
    • one: two pointers
      • left, right = 0, len(height) - 1

      • if height[left] > height[right] => the same, move the right pointer

31. Next Permutation

  • Description:

  • Solutions:
    • one:
      • check from the end of the array while increasing, mark the pivot point while the left number is lower than right

33. Search in Rotated Sorted Array

  • Description:
    • There is an integer array nums sorted in ascending order (with distinct values).
    • Prior to being passed to your function, nums is possibly rotated at an unknown pivot index k (1 <= k < nums.length) such that the resulting array is [nums[k], nums[k+1], …, nums[n-1], nums[0], nums[1], …, nums[k-1]] (0-indexed). For example, [0,1,2,4,5,6,7] might be rotated at pivot index 3 and become [4,5,6,7,0,1,2].
    • Given the array nums after the possible rotation and an integer target, return the index of target if it is in nums, or -1 if it is not in nums.
    • You must write an algorithm with O(log n) runtime complexity.
  • Solutions:
    • the runtime complexity should be limited at O(logn), so we have to use binary search to solve this question, there are following situations
      • move right
        • [1, 2, 3, 4, 0] -> target > mid > left
        • [4, 0, 1, 2, 3] -> mid < target < right
        • [2, 3, 4, 0, 1] -> target < left < mid

45. Jump Game II

  • Description:
    • Given an array of non-negative integers nums, you are initially positioned at the first index of the array.
    • Each element in the array represents your maximum jump length at that position.
    • Your goal is to reach the last index in the minimum number of jumps.
  • Solutions:
    • one: O(n) -> we can define
      • cur: the range of this loop
      • nex: the range of next loop
      • step: the step we will use
      • index: current position
      • if index <= cur: nothing else: cur = nex, step + 1
    • two: dynamic algorithm
      • dp[i+j] = min(dp[i]+1, dp[i+j])
